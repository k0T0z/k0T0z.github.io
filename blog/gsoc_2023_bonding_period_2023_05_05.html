GSoC '23 - Bonding Period: Getting started
05 May 2023

home | blogs

This blog post is related to my GSoC 2023 Project.

After 1 year of waiting, I have finally been selected for GSoC 2023! I really can't wait to work with my mentors Robert, Josh, rpjohnst, and Hugar this summer.

I decided to work on the Steam Workshop SDK/Third Party Integration idea for GSoC 2023. The idea is to implement a wrapper around the Steamworks API, and then implement User functions' APIs for ENIGMA game engine. Game developers can then use these functions (sometimes called scripts) to integrate their games with Steamworks.

This was a very important period for me, I can't say it's special, I figured out that creating the wrapper itself isn't the project. ENIGMA is used for creating games by writing EDL scripts, it's not like Unreal Engine for example, as it uses C++ mainly for creating games, but for ENIGMA, we will create the wrapper itself and then write the necessary user functions for different APIs mentioned in my proposal. This is not enough, ENIGMA uses EDL for creating games but it's written in C++ so why create a wrapper in the first place? Steam SDK is also written in C++ so I can directly go ahead and write the user functions. "I think it'd be better if you did that and did not use the abstraction" Robert replied, he meant that it will be better if I implemented the wrapper, although it is not really that useful, so I guess it's a good coding practice, who knows?

Before getting started, I must familiarize myself with ENIGMA's extension system as it is the basis of this project. I wrote a Makefile file (thanks to rpjohnst) to link the Steam lib with the executable. This Makefile file is written before getting accepted into the program, I will be using it through the summer, and will be modified if needed.
