Now, let's understand more about how ENIGMA's Async system works. From the output when firing an event it looks like this: 

    INFO: Steam ASYNC: {  }

When I run the GMS official example game, the Async output is between those parenthesis. My output looks like this for now: 

    INFO: Steam ASYNC: {  }
    INFO: leaderboard_download

I don't it is not inside the parenthesis, let's try to fix it. A new idea came to my mind to find out where the code that prints this output in the console. I remembered my old conversation with Josh, and he said that ENIGMA has a code generation system. I don't know how it works yet, but this means that the code in the main repo is not the full running code. Maybe if I can get to this generated code, I can find what I need so let's try that.

Inside ~/.enigma/Preprocessor_Environment_Editable/ directory, I used VSCode search in files feature. I love VSCode because of this one by the way. I found that there is a file called IDE_EDIT_objectfunctionality.h that has the what I was looking for: 

    variant enigma::OBJ_Obj_Steam::myevent_steam()
    {
        enigma::temp_event_scope ENIGMA_PUSH_ITERATOR_AND_VALIDATE(this);
        
        {
        show_debug_message("Steam ASYNC: "+ json_encode(async_load));
        
        }
        ;
        
        return 0;
    }

From what I see is that it uses async_load variable to get the output. This means that there is no output inside, which means that there is a problem with the new Async system I implemented. Let's try to fix it.
